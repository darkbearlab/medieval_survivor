## A. 新可遊玩角色

### 現狀
已有 `player_warrior`（戰士）和 `player_mage`（法師）兩種角色，角色選擇畫面已完成。
Player 實體目前只有一份屬性，角色間的差異靠 `characterType` 傳入 config 的修正值。

### 規劃角色

#### A1 — 聖騎士（Paladin）
- 高HP、低移速、近戰範圍攻擊（扇形/圓形 AoE）
- 被動：10 秒一次自動護盾（吸收一次傷害）
- 特色建築加成：城牆/城堡血量額外 +20%

#### A2 — 遊俠（Ranger）
- 高移速、低HP、雙發箭矢
- 被動：採集範圍自帶擴大（等同有一座採集所）
- 特色：士兵和弓箭手升級費用 -20%

#### A3 — 術士（Warlock）
- 中等屬性、召喚系（每 X 秒可召喚一隻臨時骷髏）
- 被動：每殺一個敵人回 1HP
- 特色：法師塔免費且法師攻擊力 +30%

### 實作要點
- `Player.js` 需支援角色類型決定攻擊模式（單發、扇形、雙發、AoE）
- `CharacterSelectScene` 加入第 3-4 個選項格
- config 加入每種角色的 `BASE_STATS`
- 角色特效 / HUD 小頭像可用 `generateTexture` 繪製新色調
- **相依性**：無，可獨立實作

## B. 新地圖（固定佈局 + 尋路）

### 現狀與問題
目前地圖是完全空曠的 2560×2560 平面，資源點隨機散落。
Phase 3 曾加入不可穿越地形，後來移除因為遇到**尋路問題**：
Arcade Physics 的敵人用 `physics.moveToObject` 直線移動，遇到靜態障礙物就卡住。

### 尋路方案評估

| 方案 | 複雜度 | 效果 |
|------|--------|------|
| **方案 X（建議）**：簡化迴避轉向 | 低 | 夠用但偶有卡邊角 |
| EasyStar.js（格子 A*） | 中 | 正確但每幀計算多 |
| Phaser Navmesh plugin | 高 | 最流暢，需引入外部依賴 |

**建議採方案 X**：在地圖設計上確保障礙物只構成「走廊」而非「迷宮」，
敵人偵測到正前方 60px 內有靜態物體時，往左或右偏轉 45° 繼續移動。
這樣設計避免複雜尋路，配合地圖佈局即可正確運作。

### 規劃地圖

#### B1 — 峽谷要塞（Canyon Pass）
- 地形：左右各一排岩石山脈，形成南北向走廊
- 敵人主要從南北兩端湧入，東西攻擊極少
- 玩家優勢：兩側建城牆封住走廊效益極高
- 資源：隨機散落在走廊和少數山脈缺口

#### B2 — 森林空地（Forest Clearing）
- 地形：四角和邊緣有密林（大矩形障礙），中央是空地
- 敵人必須從密林之間的缺口進入
- 玩家優勢：缺口位置固定，利於集中防禦
- 資源：多樹木資源（木材更豐富），石材較少

#### B3 — 廢墟城鎮（Ruined Town）
- 地形：散落的建築廢墟（中等矩形障礙），不規則分佈
- 敵人路徑多樣，防守難度最高
- 特色：廢墟可被建築師用少量資源「修繕」成臨時城牆
- 資源：廢墟內有額外金幣掉落物

### 實作要點
- `MapSystem.js`（新）：管理地圖設定，包含靜態障礙座標
- `BootScene.js`：障礙物用 `generateTexture` 繪製（岩石/樹/廢墟色調）
- 建造系統需過濾掉障礙物區域（`BuildingSystem._canPlace` 加入地形碰撞判斷）
- 地圖選擇在 `CharacterSelectScene` 或新增一個 `MapSelectScene`
- **相依性**：建議在 A（新角色）之後實作，因為需要測試各角色在地圖中的體驗

## C. 升級深化（掉落物、武器升級、建築升級）

### C1 — 武器升級擴充

現有 6 種武器升級（速率/彈藥/爆炸/貫穿/閃電/冰凍），每種最多 3 級。

#### 新增升級項目

**主動技能類（選到後獲得全新攻擊方式）：**
- 🌀 **旋風斬**：每 8 秒自動對周圍 80px 範圍造成 3 倍傷害（近戰系角色加成）
- 🔥 **火牆**：每 12 秒前方生成持續 3 秒的火焰牆，敵人穿越持續受傷
- ⚡ **雷霆連鎖**：攻擊命中後 30% 機率彈射至第二個目標（已有閃電升級，這是強化版）

**數值類（可疊加更多層）：**
- 現有升級從「最多 3 級」擴展到「最多 5 級」，每級效益遞減
- 加入「稀有升級」：出現機率 20%，效果是同名升級的 2 倍但標示金色

**升級協同（Synergy）：**
- 擁有「爆炸彈」+「貫穿」→ 解鎖「穿地爆彈」升級選項
- 擁有「閃電鏈」+「冰凍」→ 解鎖「雷冰爆」升級選項

### C2 — 建築升級加深

| 建築 | 現有最高 | 新增等級 | 新效果 |
|------|---------|---------|--------|
| 箭塔 | Lv2 | **Lv3** | 雙箭頭（同時射 2 發，不同目標） |
| 兵營 | Lv2 | **Lv3** | 最大士兵 +1，士兵重生時間 -30% |
| 法師塔 | Lv2 | **Lv3** | 法師 AoE 傷害 +50%，召喚速率 -20% |
| 旅店 | Lv2 | **Lv3** | 療癒範圍擴大，同時恢復玩家 HP |
| 採集所 | Lv1 | **Lv2** | 採集量 ×2，自動搬運至資源欄 |
| 維修工 | Lv1 | **Lv2** | 同時修復範圍內所有建築，修復速度 ×1.5 |
| 城牆 | Lv2 | **Lv3** | 城牆變為「尖刺牆」，接觸的敵人每秒受傷 |

### C3 — 新建築（2–3 棟）

#### 瞭望塔（Watchtower）
- 功能：擴大全地圖視野提示——當有敵人進入距離玩家 600px 內，在 HUD 邊緣顯示方向箭頭警示
- 附加：附近的箭塔射程 +20%
- 費用：木材 20 + 石材 10

#### 銀行（Treasury）
- 功能：每 30 秒產生少量金幣（被動收入），可升級增加收入頻率
- 費用：金幣 30 + 石材 20
- 特色：只能建 1 棟

#### 祭壇（Shrine）
- 功能：不可升級，使用一次性效果（消耗 15 糧食）
  - **戰鼓祭典**：士兵和法師移速 +50% 持續 20 秒
  - **豐收祭典**：立即獲得 20 糧食
  - **戰爭祭典**：所有建築攻擊力暫時翻倍 15 秒
- 費用：金幣 25 + 木材 15，使用後需 60 秒冷卻

### C4 — 掉落物多樣化

現有寶箱只出現武器/數值升級（`UpgradeChoiceScene`）。

新增掉落類型（隨機三選一時加入新池）：
- 💰 **意外之財**：直接獲得 40 金幣
- 🛠 **建築藍圖**：隨機一棟建築的費用下次建造 -50%
- 🍞 **緊急糧援**：立即獲得 20 糧食 + 糧食上限 +10
- 🏅 **英雄勛章**：所有士兵/法師 HP 永久 +10%（本局有效）
- ⚗ **神秘藥劑**：隨機效果（可能是大幅加強，也可能是小懲罰——增加趣味性）

## 已完成基礎架構（2026-03-28）

### 武器系統重構 ✓
- 新增 `CONFIG.WEAPONS`：每把武器有獨立的 `RANGE / RATE / DAMAGE_MULT / AOE / AOE_RADIUS`
- `CONFIG.CHARACTERS` 移除武器屬性，改用 `STARTING_WEAPON` 指向武器鍵
- `Player.js` 從 `CONFIG.WEAPONS[weaponKey]` 讀取攻擊屬性（`player.weaponKey` 可查詢）
- `GameScene._applyUpgradeEffect(key)` 抽出純邏輯，供 `_applyUpgrade` 和 `_applyStartingBonus` 共用

### 角色起始加成系統 ✓
- `CONFIG.CHARACTERS[key].STARTING_BONUS`（選填）支援：
  - `gold / wood / stone / food`：初始額外資源
  - `freeBuildings: string[]`：初始免費建築（直接入庫）
  - `upgrades: string[]`：初始已解鎖升級
- `GameScene._applyStartingBonus()` 在遊戲開始後立即執行

### 新增角色只需做三件事
1. 在 `CONFIG.WEAPONS` 加入該角色的武器（如有獨特攻擊方式）
2. 在 `CONFIG.CHARACTERS` 加入角色資料，填入 `STARTING_WEAPON` 和 `STARTING_BONUS`
3. 在 `BootScene.js` 用 `generateTexture` 繪製角色外觀
4. 在 `CharacterSelectScene.js` 的 `CHAR_KEYS` 加入新鍵名

## 技術備忘

### 尋路相依性圖

```
新地圖（B）
  └─ 需要：障礙物靜態物理體
       └─ 需要：敵人 AI 迴避邏輯（Enemy._avoidObstacles）
            └─ 可選：EasyStar.js 或簡化轉向
  └─ 需要：MapSystem.js
  └─ 需要：建築放置驗證擴充
```

### 角色相依性圖

```
新角色（A）
  └─ 需要：Player.js 攻擊模式系統化（目前已有部分多型）
  └─ 需要：CharacterSelectScene 擴充 UI
  └─ 可選：角色專屬被動效果 hook（GameScene 或 Player）
```

### 升級深化相依性圖

```
升級協同（C1 Synergy）
  └─ 需要：UpgradeChoiceScene 讀取已持有升級
  └─ 需要：config 加入 SYNERGY 條件定義

建築升級（C2）
  └─ 需要：各建築 upgrade() 支援 Lv3
  └─ 需要：config UPGRADE[3] 資料填入
  └─ 需要：UpgradePanel._refresh 顯示 Lv3 資訊
```
